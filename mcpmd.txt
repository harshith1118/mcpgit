

# PRD: GitHub Intelligence MCP Server

## 1. Project Overview

Build a production-style MCP (Model Context Protocol) server using Python.

The MCP server connects with the public GitHub REST API and exposes tools that allow an LLM agent to analyze GitHub users and repositories.

The server must return clean structured JSON responses.

Primary goal:
Create an MCP server that acts as a bridge between an AI agent and GitHub developer data.

# 5. Tools Implementation

The server must expose exactly these MCP tools.

---

## Tool 1: get_github_profile

Purpose:

Fetch GitHub user information.

Input:

```json
{
 "username":"string"
}
```

Example:

```
get_github_profile("torvalds")
```

API:

```
GET https://api.github.com/users/{username}
```

Return JSON:

```json
{
 "success":true,
 "data":{
    "username":"torvalds",
    "name":"Linus Torvalds",
    "followers":10000,
    "following":0,
    "public_repos":10,
    "profile_url":"..."
 },
 "error":null
}
```

Handle:

* invalid username
* API failure
* network errors

---

# Tool 2: analyze_repository

Purpose:

Analyze GitHub repository information.

Input:

```json
{
 "owner":"string",
 "repository":"string"
}
```

Example:

```
analyze_repository(
"facebook",
"react"
)
```

API:

```
GET https://api.github.com/repos/{owner}/{repository}
```

Return:

```json
{
"success":true,

"data":{

"name":"react",

"description":"UI library",

"stars":230000,

"forks":40000,

"language":"JavaScript",

"open_issues":1000,

"created_at":"date"

},

"error":null

}
```

# 8. Error Handling Requirements

Every response must follow:

```json
{
"success":boolean,
"data":object|null,
"error":string|null
}
```

Never expose raw API errors.
